# 🤖 Auto Detection System

## Обзор

Система автоматического обнаружения спрайтов использует алгоритмы компьютерного зрения для:
1. **Автоматического обнаружения** всех спрайтов на sprite sheet
2. **AI-классификации** типов объектов (персонажи, враги, боссы, тайлы, фон)
3. **Формирования сеток** с автоматическим определением параметров
4. **Поддержки динамических размеров** - не только фиксированная сетка

## Как работает

### 1. Алгоритм Flood Fill
Система сканирует изображение попиксельно и использует flood fill для поиска связанных непрозрачных областей (alpha > 10).

### 2. Фильтрация по размеру
Отсеиваются слишком маленькие (<8px) и слишком большие (>1024px) объекты.

### 3. AI-классификация

Каждый найденный спрайт классифицируется по характеристикам:

| Тип | Критерии |
|-----|----------|
| **Boss** | Очень большой (>5% изображения) |
| **Parallax** | Широкий (aspect ratio >3), сверху |
| **Tile** | Маленький (~64x64), квадратный, плотный |
| **UI** | Маленький, у краёв изображения |
| **Character** | Средний размер, слева от центра |
| **Enemy** | Средний размер, справа от центра |
| **Effect** | Маленький, низкая плотность пикселей |
| **Item** | Маленький/средний, средняя плотность |

### 4. Группировка в сетки

Система ищет паттерны одинаковых спрайтов и автоматически создаёт:
- **Origin** - координаты начала сетки
- **Cell Size** - средний размер ячейки
- **Spacing** - расстояние между ячейками
- **Columns/Rows** - количество столбцов и строк

### 5. Создание фреймов

Для каждого обнаруженного спрайта создаётся отдельный frame с:
- Точными координатами (X, Y)
- Динамическим размером (Width, Height)
- Автоматическим именем

## Использование

### Быстрый старт

1. **Загрузите изображение** (Ctrl+O)
2. **Нажмите "🤖 Auto Detect"**
3. **Готово!** Группы созданы автоматически

### Древовидное отображение групп

Группы отображаются в виде дерева по типам:

```
🎮 Characters
   └─ Player (8)
👾 Enemies  
   ├─ Enemy_Type1 (6)
   └─ Enemy_Type2 (4)
🧱 Tiles
   └─ Ground_Tiles (64)
🌄 Parallax
   └─ Background_Layer (3)
```

### Настройка после автообнаружения

После автоматического обнаружения вы можете:
- ✅ Откорректировать границы вручную
- ✅ Переименовать группы
- ✅ Изменить типы
- ✅ Перетащить сетку (Ctrl + drag)
- ✅ Добавить/удалить фреймы

## Технические детали

### AutoDetectionService.cs

Основной сервис с алгоритмами:
- `DetectSprites()` - главный метод обнаружения
- `FloodFill()` - поиск связанных областей
- `ClassifySprite()` - AI-классификация
- `GroupSpritesIntoGrids()` - формирование сеток

### ImageDataExtractor.cs

Helper для работы с пиксельными данными:
- `ExtractPixelData()` - получение RGBA массива
- `GetPixel()` - чтение цвета пикселя
- `IsTransparent()` - проверка прозрачности

### Параметры

```csharp
MinSpriteSize = 8;      // Минимальный размер спрайта
MaxSpriteSize = 1024;   // Максимальный размер
AlphaThreshold = 10;    // Порог непрозрачности (0-255)
```

## Советы

### ✅ Лучшие практики

1. **Используйте чистые sprite sheets** - без лишних объектов
2. **Прозрачный фон** - alpha = 0 для пустых областей
3. **Равномерное расстояние** между спрайтами для лучшего обнаружения сеток
4. **Группируйте по типам** при создании sprite sheet

### ⚠️ Ограничения

- Работает только с PNG/WEBP с alpha-каналом
- Требует достаточного контраста с фоном
- Может неправильно классифицировать уникальные объекты
- Сложные overlapping спрайты могут быть объединены

### 🔧 Если что-то пошло не так

1. **Проверьте прозрачность фона** - должен быть полностью прозрачным
2. **Разделите спрайты** - добавьте минимум 2px расстояния
3. **Настройте вручную** после автообнаружения
4. **Измените тип группы** если классификация неверна
